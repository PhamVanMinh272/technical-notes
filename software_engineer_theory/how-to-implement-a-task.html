Architectural Thinking Checklist — Senior Python Developer

Quick guide: clarify, design, prototype, observe.

Top‑level approach (first 10–30 minutes)

  • Clarify goal: users, success criteria, SLOs (latency, uptime).
  • Identify constraints: timeline, infra (cloud/on‑prem), team skills.
  • Define failure modes: slow deps, DB down, malformed inputs.

Architecture dimensions

DimensionKey questionDesign hint
StateStateless or stateful?Prefer stateless services; use durable storage for consistency.
ScaleQPS and latency targets?Cache, async workers, sharding when needed.
ResilienceHow to handle failures?Timeouts, retries, circuit breakers, graceful degradation.
ObservabilityHow will we detect issues?Metrics, structured logs, distributed traces, alerts.

Practical steps to implement

  1. Sketch a diagram (10 minutes): components, data flow, failure boundaries.
  2. Write acceptance criteria & NFRs: SLOs, MTTR targets, security needs.
  3. Prototype the riskiest path: validate assumptions, add observability hooks.
  4. Define contracts: API schemas, message formats, idempotency rules.
  5. Plan rollout: canary, feature flags, automated rollback.

Risks & tradeoffs

  • Over‑engineering: avoid premature microservices; prefer modular monoliths for fast delivery.
  • Operational debt: instrument and automate runbooks early.
  • Performance vs resilience: defensive checks add latency—measure and tune.

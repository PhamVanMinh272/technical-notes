Q1. What is a decorator and how is it typically used Short answer
**Answer:** A decorator is a callable that takes a function or class and returns a modified function or class. It's commonly used to add behavior like logging, access control, or caching without changing the original function body.
Example: `@functools.wraps` preserves metadata; `@cache` adds memoization.
Q2. How do generators differ from normal functions Key idea
**Answer:** Generators use `yield` to produce a sequence of values lazily, maintaining state between yields. They are memory-efficient for large sequences and can be iterated like iterators.
Use `next()` or `for` loops; `yield from` delegates to subgenerators.
Q3. What is a context manager and how to implement one Pattern
**Answer:** A context manager defines setup and teardown around a block using `with`. Implement via a class with `__enter__` and `__exit__` or a generator decorated with `@contextlib.contextmanager`.
`__exit__` handles exceptions; return value of `__enter__` is bound to the `as` target.
Q4. Define closure and give a use case Concept
**Answer:** A closure is a function that captures variables from its enclosing scope, preserving them after the outer function returns. Use closures for factory functions or to maintain private state without classes.
Captured variables are read-only unless declared `nonlocal`.
Q5. What are metaclasses and when to use them Advanced
**Answer:** Metaclasses control class creation (they are the class of a class). Use them to enforce class invariants, auto-register subclasses, or modify attributes at class creation time.
Default metaclass is `type`; override `__new__` or `__init__` in a metaclass.

Q6. How does Python garbage collection work Mechanism
**Answer:** CPython uses reference counting plus a cyclic garbage collector to detect and collect reference cycles. Objects are freed when reference count drops to zero; cycles are handled by the GC generations.
Use `gc` module to inspect or tune collection; watch for __del__ interactions.
Q7. Name two profiling tools and when to use them Tools
**Answer:** `cProfile` for deterministic function-level profiling; `line_profiler` for line-by-line hotspots. Use `memory_profiler` to track memory usage.
Combine profiling with representative workloads for accurate results.
Q8. Give one optimization technique for Python code Practical
**Answer:** Use algorithmic improvements first (better data structures), then reduce allocations (reuse objects), use built-in functions and comprehensions, and consider C extensions or PyPy for heavy workloads.
Profile before optimizing; micro-optimizations often have limited impact.

Q9. When is threading appropriate in Python Use case
**Answer:** Threading is suitable for I/O-bound tasks where the Global Interpreter Lock (GIL) is not a bottleneck, such as network requests or file I/O. For CPU-bound tasks, prefer multiprocessing.
Use `concurrent.futures.ThreadPoolExecutor` for simpler thread pools.
Q10. What does asyncio provide and when to use it Async
**Answer:** `asyncio` provides an event loop and async/await syntax for cooperative concurrency. Use it for high-concurrency I/O-bound applications like web servers or clients.
Combine with `aiohttp` for async HTTP; avoid blocking calls inside async code.
Q11. How does multiprocessing differ from threading Difference
**Answer:** Multiprocessing runs separate Python processes with independent memory, bypassing the GIL and enabling true parallel CPU-bound work. Communication uses IPC like queues or pipes.
Process startup cost is higher; use `multiprocessing.Pool` for worker pools.

Q12. Best practice for exception handling Pattern
**Answer:** Catch specific exceptions, avoid bare `except:`, clean up resources in `finally` or use context managers, and fail fast with clear error messages. Log exceptions with context.
Use custom exception types for domain-specific errors.
Q13. Unit testing vs integration testing Compare
**Answer:** Unit tests verify small, isolated units (functions/classes) often with mocks. Integration tests verify interactions between components and external systems. Both are essential.
Use `pytest` for expressive tests and fixtures; run integration tests in CI with realistic environments.
Q14. How to test async code with pytest Technique
**Answer:** Use `pytest` with `pytest-asyncio` or `anyio` to write `async def` tests and run them on an event loop. Use fixtures to set up async resources.
Mark tests with `@pytest.mark.asyncio` or use `asyncio` fixtures provided by plugins.

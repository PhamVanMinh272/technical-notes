Aspect Definition Practical signals
Maintainable Easy to read, understand, and change without fear of regressions. Consistent style, clear naming, code reviews, linters, low cyclomatic complexity, clear boundaries.
Scalable Grows gracefully with users, data, and features without major rewrites. Horizontal scaling paths, stateless services, efficient DB access, caching, idempotent ops, back-pressure.
Reliable Functions correctly and consistently under expected conditions. SLOs/SLAs, health checks, retries with jitter, circuit breakers, graceful degradation, error budgets.
Testable Designed so automated tests are easy to write and run. DI-friendly, pure functions, seamful design, fast unit tests, CI pipelines, contract & E2E tests.
Performant Uses resources efficiently with quick responses. Profiling, APM traces, O(n) awareness, async where needed, pagination, lazy loading, memory discipline.
Wellâ€‘documented Clear docs for developers and future contributors. README with setup/run, ADRs, API specs, runbooks, onboarding guides, comments where it adds context.
Secure Protects data and resists common vulnerabilities. Least privilege, secret management, dependency scanning, input validation, TLS, authn/z, threat modeling.
Modular Loosely coupled, highly cohesive components that are reusable. Clear interfaces, bounded contexts, small surfaces, versioned contracts, dependency inversion.
